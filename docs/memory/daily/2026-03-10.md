## Inefficiency and lessons

- **Many deployments, incomplete results:** Multiple deploys and a large amount of time/energy were spent; the agent page is still not fully functional. Changes were broad (theme refactor, many files) rather than minimal and targeted.
- **Assumptions:** Some fixes were assumed to work without verification on the actual devices/viewports (e.g. header on Mac, Monaco on phone). Verification should happen on real devices and viewports before considering a fix done.
- **Strategy for tomorrow:** Do not repeat this pattern. Proceed one fix at a time. Verify each fix (e.g. on Mac, on phone) before moving to the next. Do not destroy existing behavior. Only make the minimal changes needed to achieve 100% solidified agent page functionality and UI.
